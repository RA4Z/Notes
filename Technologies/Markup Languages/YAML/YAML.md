Yet Another Markup Language is a human-readable data serialization language used primarily for configuration files, data exchange and documentations

### Characteristics
- **Human-readable:** YAML is designed to be easily understood by people, using a clean syntax that relies on indentation 
- **Data serialization:** It's used to convert data structures into a format that can be stored, transmitted and reconstructed
- **Configuration files:** A very common use is in configuration files for software and infrastructure

### Common use cases
- **Configuration:** Setting up configurations for applications, such as in the Home Assistant platform
- **[[Pipeline|CI/CD pipelines]]:** Specifying the steps and jobs for continuous integration and continuous delivery pipelines in services like GitLab and GitHub Actions
- **Containerization:** Defining deployments for systems like Kubernetes
