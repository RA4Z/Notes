[[Git]] commands to revert changes, clean up the directory or manage temporary states

- `git reset`: Resets the [[HEAD]] to a specific state, potentially modifying the staging area and working directory
- `git revert`: Creates a new commit that undoes the changes of a previous commit
- `git restore`: Restores files in the working directory or staging area
- `git clean`: Removes untracked files from the working directory
- `git stash`: Temporarily saves changes that are not ready to be committed
